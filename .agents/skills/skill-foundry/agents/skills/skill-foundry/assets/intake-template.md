# Skill intake template

## Workflow summary
What repeated workflow are we considering turning into a skill?

## Intended outcome
What should the skill reliably produce or improve?

## Trigger phrases
What are likely user prompts that should activate the skill?

## Out-of-scope prompts
What should *not* activate the skill?

## Required knowledge
What domain knowledge, policy, or conventions must the skill encode?

## Tools and dependencies
What tools, scripts, or references are needed?

## Pattern choice
- tool-wrapper
- generator
- reviewer
- inversion
- pipeline

## Risks
What could go wrong if the skill is too vague, too rigid, or outdated?
