# Description heuristics

The description is routing logic. It should explain both what the skill does and when it should be used.

## Good description characteristics
- Names the core task clearly
- Includes likely user intents or trigger phrases
- States scope boundaries
- Avoids vague words like "help" or "general"
- Avoids being so narrow that only one exact phrasing can trigger it

## Rewrite tips
- Replace internal jargon with terms users actually say
- Add one sentence about when not to use the skill
- Mention the output type when useful
- Mention the domain or artifact only when it truly narrows scope

## Examples
Too vague:
- "Writing help"

Too narrow:
- "Q4 2025 product launch email sequence for premium segment"

Better:
- "Create high-converting product launch and lifecycle marketing emails. Use when the user needs email copy with clear structure, audience targeting, subject lines, and calls to action. Do not use for general long-form blog writing or brand strategy."
