# Skill anti-patterns

## The god skill
One massive skill with many unrelated jobs. Split by trigger boundary or workflow.

## The one-off prompt disguised as a skill
If the work is not repeated, do not create a skill.

## The stale success case
A skill that worked once months ago but has not been benchmarked since major model updates.

## The invisible trigger
A skill description that uses vague or obscure wording and rarely activates.

## The over-constrained relic
A skill that forces old rigid steps even after baseline model behavior has improved.

## Reference overload
Putting every example and note into `SKILL.md` instead of using `references/` and `assets/`.
